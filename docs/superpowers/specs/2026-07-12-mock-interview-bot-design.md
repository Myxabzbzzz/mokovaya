# Дизайн — «Моковая» (Telegram-бот для мок-собеседований)

## 1. Обзор

Telegram-бот для тренировки алгоритмических собеседований между двумя живыми людьми. Бот не проводит собеседование сам — он подбирает пару (интервьюер ↔ кандидат) и передаёт контакты для связи; сам разговор и решение задачи проходят вне бота, по договорённости пользователей.

Ключевая идея, которую должен понять пользователь при первом запуске: это не два разных типа аккаунтов, а **две роли, между которыми свободно переключаешься каждый раз** — сегодня ты интервьюируешь, завтра тебя интервьюируют.

## 2. Онбординг и объяснение ролей

Текст при первом `/start`:

> **Добро пожаловать в Моковую 👋**
> Здесь можно тренировать алгоритмические собеседования вживую — с реальным человеком, а не с ботом.
>
> Как это работает:
> 🎤 **Ты — интервьюер.** Задаёшь задачу и оцениваешь решение — как настоящий интервьюер в компании.
> 💻 **Ты — кандидат.** Решаешь задачу под наблюдением, как на реальном собеседовании.
>
> Роль выбираешь каждый раз заново — можно сегодня быть интервьюером, завтра кандидатом. Многие делают оба захода подряд: сначала спрашивают сами, потом отвечают.
>
> Бот подбирает пару и даёт контакт для связи — дальше вы сами договариваетесь о времени и площадке для звонка.

Главное меню — два пункта:
- 🎤 **Хочу собеседовать** (встать в очередь как интервьюер)
- 💻 **Хочу пройти собеседование** (встать в очередь как кандидат)

## 3. Регистрация

Один раз, сразу после приветствия:

1. Бот спрашивает имя (можно предложить имя из профиля Telegram по умолчанию, с возможностью изменить).
2. Бот открытым вопросом спрашивает контакт: **«Как с тобой можно связаться? Напиши Telegram-юзернейм или номер телефона»**. Пользователь сам вписывает текстом, что ему удобно (юзернейм или телефон). Валидация формата не требуется — что ввёл, то и передаётся партнёру при мэтче.

После этого пользователь сразу попадает в главное меню.

## 4. Очередь и мэтчинг

- Нажатие «Хочу собеседовать» → пользователь встаёт в очередь `interviewer`. Если в очереди `candidate` уже кто-то есть — матчим сразу с первым пришедшим (FIFO). Если очередь `candidate` пуста — пользователь ждёт, бот показывает «Ищем кандидата... Можно отменить поиск».
- Симметрично для «Хочу пройти собеседование» → очередь `candidate`.
- Пользователь не может стоять в обеих очередях одновременно — выбор роли перед постановкой в очередь это гарантирует естественным образом.
- При мэтче — обоим участникам приходит сообщение с именем и контактом партнёра: «Найден партнёр! Имя: …, Контакт: … Договоритесь о времени сами.» Оба удаляются из очереди (статус `matched`).
- Отмена ожидания доступна в любой момент кнопкой «Отменить поиск».
- Тайм-аута ожидания на MVP нет — пользователь ждёт, пока не отменит сам или не найдётся пара.
- Встретиться в паре с самим собой технически невозможно: для мэтча нужны два разных пользователя из противоположных очередей — это следствие модели, а не отдельная проверка.

### Обработка граничных случаев
- Повторная попытка встать в очередь при уже активном ожидании → «Ты уже в очереди, отмени поиск, если хочешь передумать».
- Попытка отменить поиск, когда пользователь не в очереди → «Ты не в очереди».

## 5. Стек и данные

- **Стек:** Python + aiogram + PostgreSQL, по аналогии с предыдущими проектами (Well Prepared bot, ExamPrepBot). Модели через SQLAlchemy + alembic-миграции для консистентности с остальными проектами.
- **Таблицы:**
  - `users`: `telegram_id`, `name`, `contact` (текст, свободный формат), `created_at`
  - `queue`: `user_id`, `role` (`interviewer` / `candidate`), `joined_at`, `status` (`waiting` / `matched` / `cancelled`)
  - `matches`: `interviewer_id`, `candidate_id`, `matched_at` — журнал для истории, не требуется для логики мэтчинга напрямую

## 6. Вне рамок MVP (сознательно не делаем сейчас)

- Учёт языка программирования/уровня при мэтчинге — матчим только по роли.
- Расписание/слоты по времени — только онлайн-очередь «прямо сейчас».
- Проведение самого интервью внутри бота (код-редактор, видео/аудио) — всё вне бота, по контактам.
- Тайм-аут очереди, автоотмена.
- Рейтинги/фидбек после собеседования.
